# バッファタスク集

「講師待ち」や「エラーで詰まった」ときに、時間を有効活用するためのタスク集です。

---

## レベル1: AIと対話（いつでも可能）

講師を待っている間、Claudeに以下のような質問をしてみましょう。

### アプリの改善点を聞く

```
このアプリの要件定義/コードを読んで、改善点を10個挙げてください。
優先度が高いものから順に教えてください。

[要件定義またはコードを貼り付け]
```

### コードを初心者向けに説明してもらう

```
以下のコードを、プログラミング初心者にも分かるように、
1行ずつ日本語で説明してください。

[コードを貼り付け]
```

### アプリのアイコンデザインを考える

```
私のアプリは[アプリの説明]です。
このアプリにふさわしいアイコンのデザイン案を3つ提案してください。
それぞれのデザインのコンセプトも説明してください。
```

### 画面遷移を言語化する

```
私のアプリの画面遷移を、初心者でも分かるように図解してください。

画面:
1. ホーム画面
2. タスク追加画面
3. タスク詳細画面

遷移:
- ホームから追加ボタンを押すと追加画面へ
- 追加画面で保存するとホームに戻る
- ホームのタスクをタップすると詳細画面へ
```

## レベル2: リサーチ（個人ワーク）

自分でリサーチして、アプリのヒントを見つけましょう。

### Perplexityで競合アプリを調べる

**タスク**: Perplexityで競合アプリを3つ調べる

**検索例**:
```
[あなたのアプリのジャンル] おすすめアプリ 3選
例: タスク管理アプリ おすすめ 3選
```

**調べること**:
- どんな機能があるか
- ユーザーのレビュー（良い点・悪い点）
- 自分のアプリとの違い

**Notionにまとめる内容**:
| アプリ名 | 主な機能 | 良い点 | 悪い点 | 自分のアプリとの違い |
|---------|---------|--------|--------|---------------------|
|         |         |        |        |                     |

### App Storeで似たアプリのレビューを読む

**タスク**: App Storeで似たアプリのレビューを10件読む

**読むポイント**:
- ★1〜2の低評価レビュー → ユーザーの不満を知る
- ★5の高評価レビュー → ユーザーが何を求めているかを知る

**Notionにまとめる内容**:
- よく見る不満（TOP3）
- よく見る満足ポイント（TOP3）
- 自分のアプリに活かせること

### ターゲットユーザーへのインタビュー質問を考える

**タスク**: もし実際にユーザーにインタビューするなら、何を聞くか考える

**質問例**:
1. 今、[課題]についてどう対処していますか?
2. 既存の[類似アプリ]で不満に思うことはありますか?
3. 理想的なアプリがあるとしたら、どんな機能が欲しいですか?
4. このアプリを1日のどのタイミングで使いたいですか?
5. このアプリにいくらまでなら払えますか?

**Notionにまとめる**: インタビュー質問リスト（10個）

## レベル3: ドキュメント作成

自分用のマニュアルやナレッジベースを作りましょう。

### Notionに「自分用マニュアル」を書く

**タスク**: Notionに「[アプリ名] 開発マニュアル」ページを作る

**書く内容**:
- プロジェクトのセットアップ手順
- よく使うコマンド（`npm start`, `eas build` など）
- エラーが出たときの対処法
- よく忘れること（Supabaseの環境変数設定、など）

### 今日学んだことを自分の言葉でまとめる

**タスク**: Notionに「学習ログ」ページを作り、今日学んだことを書く

**書く内容**:
- 日付
- 今日やったこと
- 学んだこと（技術的なこと、気づいたこと）
- 次回やること

**例**:
```
## 2026-02-11

### やったこと
- Day 2のToDoアプリを完成させた
- SupabaseのRLSを設定した

### 学んだこと
- `supabase.from('tasks').select()` でデータを取得できる
- RLSを設定しないと他のユーザーのデータも見えてしまう
- Expo Routerの `<Link>` で画面遷移ができる

### 次回やること
- Day 3の自分アプリの要件定義を完成させる
- Claudeにペルソナをレビューしてもらう
```

### プロンプト集に今日うまくいったプロンプトを追加

**タスク**: Notionに「Claudeプロンプト集」ページを作る

**書く内容**:
- 今日Claudeに投げたプロンプト
- その結果（良かった点・悪かった点）
- 改善案

**例**:
```
### プロンプト: コードレビューを依頼

**投げたプロンプト**:
「以下のコードをレビューして、改善点があれば教えてください。」

**結果**:
- 良かった: 具体的な改善案をもらえた
- 悪かった: 「なぜその改善が必要か」の理由が薄かった

**改善案**:
「以下のコードをレビューして、改善点とその理由を詳しく教えてください。」
```

## レベル4: 実装練習

手を動かして、スキルを磨きましょう。

### Day 2のToDoアプリに新機能を追加

**タスク例**:
- [ ] タスクの編集機能を追加
- [ ] タスクの削除機能を追加
- [ ] タスクの検索機能を追加
- [ ] カテゴリ分類機能を追加

**やり方**:
1. Claudeに「ToDoアプリに[機能]を追加したい。どうすればいいですか?」と聞く
2. 提案されたコードを実装
3. 動作確認
4. エラーが出たらClaude に相談

### UIの色やレイアウトを変更してみる

**タスク例**:
- [ ] ボタンの色を変える
- [ ] フォントサイズを大きくする
- [ ] 画面の背景色を変える
- [ ] タスクの一覧をカードデザインにする

**やり方**:
1. StyleSheetの数値を変更してみる
2. `backgroundColor`, `fontSize`, `padding` などを試す
3. 変更したら `npm start` でリロードして確認

**参考**: React Nativeのスタイルは CSS に似ていますが、少し違います。
- `background-color` → `backgroundColor`
- `font-size` → `fontSize`
- 単位は `px` ではなく数値のみ（例: `fontSize: 20`）

### 新しいページを追加してみる

**タスク例**:
- [ ] 「設定」ページを追加
- [ ] 「プロフィール」ページを追加
- [ ] 「ヘルプ」ページを追加

**やり方**:
1. `app/` フォルダに新しいファイルを作成（例: `app/settings.tsx`）
2. Day 2で学んだ Expo Router の知識を使って、`<Link>` で画面遷移
3. ページに簡単なテキストを表示してみる

**Claudeへの質問例**:
```
Expo Routerで「設定」ページを追加したいです。
以下の構成でどうすればいいか教えてください:

- ホーム画面に「設定」ボタンを追加
- ボタンを押すと設定画面に遷移
- 設定画面には「通知ON/OFF」のスイッチを表示
```

## どのタスクから始めるべき?

### 講師を待っている間（5分〜10分）
→ **レベル1: AIと対話**

### 講師を待っている間（30分以上）
→ **レベル2: リサーチ** または **レベル3: ドキュメント作成**

### エラーで詰まって気分転換したい
→ **レベル4: 実装練習**（Day 2のアプリで遊ぶ）

### 自分のアプリの方向性を見直したい
→ **レベル2: リサーチ**（競合調査・レビュー読み）

---

## Tips

- **完璧を目指さない**: バッファタスクは「待ち時間の有効活用」が目的です。60%の完成度でOK。
- **Notionにまとめる**: 後で見返せるように、必ずNotionにメモを残しましょう。
- **Claudeを壁打ち相手に**: 一人で考えるより、Claudeに話しかけるほうが思考が整理されます。
