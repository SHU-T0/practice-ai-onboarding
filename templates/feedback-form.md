# TestFlightフィードバックフォーム

TestFlightでアプリを配信したら、ユーザーからフィードバックを集めましょう。
以下の質問項目をGoogleフォームに設定してください。

---

## Googleフォームの質問一覧

### 1. テストしているアプリ名

**質問タイプ**: 記述式（短文）

**質問文**:
```
今回テストしているアプリの名前を教えてください。
```

---

### 2. テスターの名前（任意）

**質問タイプ**: 記述式（短文）

**質問文**:
```
お名前を教えてください（匿名でも構いません）
```

---

### 3. 第一印象

**質問タイプ**: 選択式（5段階）

**質問文**:
```
アプリを起動したときの第一印象を教えてください。
```

**選択肢**:
- 5: とても良い
- 4: 良い
- 3: 普通
- 2: あまり良くない
- 1: 良くない

---

### 4. 使いやすさ

**質問タイプ**: 選択式（5段階）

**質問文**:
```
アプリの操作は分かりやすかったですか？
```

**選択肢**:
- 5: とても分かりやすい
- 4: 分かりやすい
- 3: 普通
- 2: 少し分かりにくい
- 1: 分かりにくい

---

### 5. デザインの見やすさ

**質問タイプ**: 選択式（5段階）

**質問文**:
```
文字やボタンの大きさ、色使いは見やすかったですか？
```

**選択肢**:
- 5: とても見やすい
- 4: 見やすい
- 3: 普通
- 2: 少し見にくい
- 1: 見にくい

---

### 6. 良かった点

**質問タイプ**: 記述式（段落）

**質問文**:
```
このアプリの良かった点を教えてください。
（操作性、デザイン、機能など、どんなことでも構いません）
```

---

### 7. 改善してほしい点

**質問タイプ**: 記述式（段落）

**質問文**:
```
「ここをもっとこうしてほしい」という点があれば教えてください。
```

---

### 8. バグ・不具合

**質問タイプ**: 記述式（段落）

**質問文**:
```
使っていて困ったこと、おかしな動作があれば教えてください。
（例: ボタンを押しても反応しない、画面が真っ白になる、など）
```

---

### 9. 使い続けたいか

**質問タイプ**: 選択式（5段階）

**質問文**:
```
このアプリを今後も使い続けたいと思いますか？
```

**選択肢**:
- 5: ぜひ使いたい
- 4: 使いたい
- 3: どちらとも言えない
- 2: あまり使いたくない
- 1: 使いたくない

---

### 10. その他コメント

**質問タイプ**: 記述式（段落）

**質問文**:
```
その他、何かあればご自由にお書きください。
```

## Googleフォームの設定手順

1. Google Formsにアクセス: https://forms.google.com/
2. 「空白のフォーム」を選択
3. 上記の質問を順番に追加
4. フォームのタイトルを「[アプリ名] TestFlightフィードバック」に設定
5. 「送信」をクリックして、リンクをコピー
6. TestFlightの配信メッセージにリンクを貼り付け

## フィードバック分析のClaudeプロンプト例

フィードバックが集まったら、Claudeに分析を依頼しましょう：

### プロンプト例1: 全体の傾向を掴む

```
以下はTestFlightで集めたフィードバックです。
この内容を読んで、以下を分析してください：

1. 全体的な評価の傾向（5段階評価の平均など）
2. 最も多く挙げられた「良かった点」
3. 最も多く挙げられた「改善点」
4. 優先的に対応すべき課題（TOP3）
5. ユーザーの継続利用意向（使い続けたいか）

---

[ここにGoogleフォームの回答をコピペ]
```

### プロンプト例2: バグの優先度を決める

```
以下はTestFlightで報告されたバグ・不具合の一覧です。
それぞれのバグに優先度をつけて、対応順序を教えてください。

優先度の判断基準：
- 高: アプリが使えなくなる、データが消えるなど致命的
- 中: 一部の機能が使えない、操作に支障がある
- 低: 見た目の問題、軽微な違和感

---

[ここにバグ報告をコピペ]
```

### プロンプト例3: 次のアップデートで何をすべきか

```
以下はTestFlightで集めたフィードバックです。
このフィードバックを踏まえて、次のアップデート（v1.1）で対応すべき項目を提案してください。

条件：
- 1週間以内に実装できる範囲
- ユーザー満足度への影響が大きいもの
- Must / Should / Could で優先順位をつける

---

[ここにGoogleフォームの回答をコピペ]
```

## フィードバックの活用フロー

```
1. TestFlightでアプリ配信
   ↓
2. Googleフォームでフィードバック収集
   ↓
3. Claudeで分析・優先度づけ
   ↓
4. Notionに改善項目をまとめる
   ↓
5. 次のアップデートで対応
   ↓
6. 再びTestFlightで配信（v1.1, v1.2...）
```

---

## Tips

- **フィードバックは5人以上から集める**: 1〜2人だと偏りが出ます
- **「使い続けたいか」が最重要**: この数字が低い場合、根本的な見直しが必要
- **バグはスクリーンショット付きで**: 可能なら画面キャプチャも送ってもらう
- **ポジティブな意見も大事に**: モチベーション維持のため、良かった点もちゃんと読む
