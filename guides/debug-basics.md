# デバッグ基礎ガイド

## デバッグとは？

アプリが期待通りに動かないとき、原因を見つけて修正すること。
プログラミングの50%はデバッグです。

## エラーの読み方（最重要！）

1. **赤いエラーメッセージの最後の行**を見る → ここに原因が書いてある
2. そのままコピーしてAIに貼り付ける
3. AIが修正方法を教えてくれる

## console.log デバッグ

データの中身を確認する最も簡単な方法:

```typescript
// データが取得できているか確認
const { data, error } = await supabase.from('todos').select('*')
console.log('取得したデータ:', data)
console.log('エラー:', error)

// ボタンが押されたか確認
const handlePress = () => {
  console.log('ボタンが押されました')
  console.log('入力値:', inputText)
}
```

Cursor/Expoのターミナルに出力される。

## よくあるエラーパターン

### 1. SyntaxError（文法エラー）

- 原因: カッコの閉じ忘れ、セミコロンの漏れ等
- 対処: エラーメッセージをそのままAIに貼る

### 2. TypeError: undefined is not an object

- 原因: 存在しないデータにアクセスしようとした
- 対処: console.logでデータの中身を確認

### 3. Network Error / CORS

- 原因: Supabaseへの接続設定ミス
- 対処: .env.localのURLとキーを確認

### 4. 画面が真っ白

- 原因: コンポーネントのreturn文でエラー
- 対処: 最小限のコードに戻して少しずつ追加

## AI→ペア→講師の3段階質問ルール

1. **まずAIに聞く**: エラーメッセージをCursorのチャットに貼り付ける
2. **ペアに相談**: 隣の人に画面を見せて「同じエラーになった？」と聞く
3. **講師に質問**: 上記2つで解決しない場合のみ講師に質問

## 質問テンプレート

```
【やりたいこと】
〇〇の機能を実装したい

【現状】
〇〇のコードを書いた

【困っていること】
以下のエラーが出る:
[エラーメッセージをペースト]
```
