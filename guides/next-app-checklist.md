# 「次のアプリを作るとき」チェックリスト

この研修で学んだことを活かして、自分で新しいアプリを作るためのチェックリスト。

## Phase 1: 企画（1-2日）

- [ ] ワンライナーを書く: [誰]が[何の課題]を[どう解決する]アプリ
- [ ] ターゲットユーザーを1人具体的にイメージ
- [ ] 競合アプリを3つ調査（Perplexity）
- [ ] MVP機能をMoSCoW法で仕分け
- [ ] 画面3つ以内に絞る
- [ ] テーブル2つ以内に絞る

## Phase 2: 環境構築（半日）

- [ ] Expo スタータープロジェクトをコピー
- [ ] app.jsonのbundleIdentifierを設定
- [ ] Supabaseプロジェクトを作成
- [ ] テーブルをGUIで作成
- [ ] .env.localにAPIキーを設定
- [ ] `npx expo start`で起動確認
- [ ] GitHubにリポジトリ作成 → 初回push

## Phase 3: 実装（1-2週間）

- [ ] ホーム画面を作る
- [ ] データの読み出し（SELECT）を実装
- [ ] データの追加（INSERT）を実装
- [ ] ログイン機能を実装（Supabase Auth）
- [ ] RLSを設定
- [ ] 残りの画面を作る
- [ ] UIを整える

## Phase 4: テスト・リリース（2-3日）

- [ ] 中間ビルド（eas build）を通す
- [ ] TestFlightで配布
- [ ] 5人以上にテストしてもらう
- [ ] フィードバックを収集（Googleフォーム）
- [ ] フィードバックをClaudeで分析
- [ ] 優先度の高い改善を実施
- [ ] 最終ビルド → TestFlight再配布

## 困ったときのリソース

- Cursor のチャットにエラーを貼る
- Claude Pro で相談
- Supabase ドキュメント: https://supabase.com/docs
- Expo ドキュメント: https://docs.expo.dev
- Slack #ai-builders で質問
- 週次もくもく会で相談

## やってはいけないこと

- ❌ 最初から完璧を目指す（まずMVPを出す）
- ❌ 画面を4つ以上作る（スコープ膨張）
- ❌ AIの提案をそのまま実行（理解してから実行）
- ❌ APIキーをコードに直接書く
- ❌ 1週間以上ビルドしない（早めにビルドエラーを発見）
