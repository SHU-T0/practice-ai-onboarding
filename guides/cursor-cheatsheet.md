# Cursor操作早見表

## 基本操作

| 操作 | ショートカット | 説明 |
|------|------------|------|
| Tab補完 | Tab | AIが次に書くべきコードを提案 |
| インライン編集 | Cmd+K | 選択したコードをAIに書き換えてもらう |
| チャット | Cmd+L | サイドバーでAIとチャット |
| Composer | Cmd+I | 複数ファイルにまたがる変更をAIに依頼 |

## よく使うプロンプト例（営業メンバー向け）

- 「このエラーを修正してください: [エラーメッセージをペースト]」
- 「ボタンを押したら〇〇する機能を追加してください」
- 「このページのデザインをもっときれいにしてください」
- 「一覧画面を作ってください。データは△△です」
- 「ログイン機能を追加してください」
- 「このコードを日本語でコメント付きで説明してください」

## Cursorの4つのモード使い分け

1. **Tab補完**: コードの一部を書くとAIが続きを提案。Tabで採用、Escで無視
2. **Cmd+K (インライン)**: コードの一部を選択して「〇〇に変更して」と指示
3. **Cmd+L (チャット)**: コードについて質問したり、新しい機能のコードを生成
4. **Cmd+I (Composer)**: 複数ファイルにまたがる大きな変更を一括で依頼

## コンテキスト機能（@メンション）

チャットやComposerで `@` を入力すると、AIに追加の情報を渡せます。

| コマンド | 説明 | 使い方 |
|---------|------|--------|
| `@ファイル名` | 特定のファイルを参照させる | 「@App.tsx のエラーを修正して」 |
| `@codebase` | プロジェクト全体を検索 | 「@codebase でSupabaseを使っている箇所は？」 |
| `@web` | Web検索して最新情報を取得 | 「@web Expo SDK 52の変更点」 |
| `@docs` | 公式ドキュメントを参照 | 「@docs Expo Routerの使い方」 |

> **研修では**: `@ファイル名` だけ覚えればOK。複数ファイルを指定すると、ファイル間の関係を理解した上で回答してくれます。

## AI設定のカスタマイズ

Cursorには、AIの挙動をプロジェクトに合わせて調整する仕組みがあります。
詳細は [CursorのAI設定ガイド](./cursor-ai-settings.md) を参照してください。

| 設定 | 概要 | 研修での扱い |
|------|------|------------|
| **モデル選択** | 使うAIモデルを切り替える | Claude Sonnetを標準使用 |
| **Rules** | プロジェクトのルールをAIに事前設定 | Cursor Settingsで設定済み |
| **Agent Mode** | AIが自律的に複数ステップを実行 | Day 4以降で活用 |
| **MCP** | AIに外部サービスへのアクセス権を付与 | 講師が必要に応じて設定 |

## トラブルシューティング

- AIの提案が間違っている場合: 「違います。〇〇のようにしてください」と具体的に修正指示
- エラーが出た場合: エラーメッセージをそのままチャットに貼り付ける
- 何をしていいかわからない場合: 「次に何をすべきか教えてください」
- AIの回答が的外れな場合: `@ファイル名` で関連ファイルを明示して再質問
