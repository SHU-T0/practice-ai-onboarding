## ペアプログラミングの運用ルール

実装時間中はペアプログラミングを推奨。

### 役割と交代
- **ドライバー**: Cursorを操作する人
- **ナビゲーター**: AIへのプロンプトを考える人、画面を見てフィードバックする人
- **15分ごとに交代**（タイマー推奨）

### ルール
- ナビゲーターは口頭で指示。ドライバーのキーボードを触らない
- 困ったらまずAIに質問 → ペアで議論 → Slackで相談
- 「教える」のではなく「一緒に考える」スタンス

## Day 9（木）: 機能拡張 + 中間ビルド

### 午前: 機能拡張

追加機能の実装:
- 検索・フィルタリング
- ソート機能
- お気に入り機能
- 画像アップロード（必要な場合）

### 午後: 中間ビルド

**なぜ中間ビルドが必要か？**
- 本番環境でのみ発生するエラーを早期発見
- TestFlight配布前のリハーサル
- ビルドエラーの解決に時間がかかる場合がある

### 中間ビルドチェックリスト

#### 1. ビルド前の確認

- [ ] `app.json` の設定が正しい
- [ ] `bundleIdentifier` がユニーク
- [ ] `.env.local` が.gitignoreに含まれている
- [ ] Supabase URLとAPIキーが正しい

#### 2. ビルド実行

```bash
eas build --platform ios --profile production
```

#### 3. ビルド成功の確認

- [ ] ビルドが完了（15-30分）
- [ ] エラーがない
- [ ] App Store Connectにアップロードされている

#### 4. TestFlight配布（任意）

- [ ] TestFlightで自分だけに配布
- [ ] 実機でアプリが起動する
- [ ] ログインできる
- [ ] データ操作ができる

#### 5. ビルドエラー時の対処

**エラー1: "No Provisioning Profile"**
```bash
eas credentials
```
で認証情報を再生成

**エラー2: "Bundle identifier already exists"**
`app.json` の `bundleIdentifier` を変更

**エラー3: ".env.local not found"**
環境変数をEASに設定:
```bash
eas secret:create --name EXPO_PUBLIC_SUPABASE_URL --value "https://..."
eas secret:create --name EXPO_PUBLIC_SUPABASE_ANON_KEY --value "..."
```

## トラブルシューティング

### スケジュールが遅れている場合

**判断基準**: Day 9終了時点で50%未満

**対策**:
1. **スコープ削減**: Should/Could機能を削除
2. **Slackで相談**: 技術的サポートを受ける
3. **ペア開発**: 他の受講生と協力

### モチベーションが下がった場合

- メンターと1on1
- 小さな成功体験を作る（1機能ずつ完成させる）
- 他の受講生のアプリを見てインスピレーション

### 技術的に詰まった場合

1. **Claudeに聞く**: エラーメッセージをコピペ
2. **メンターに聞く**: 相談の順番を待つ
3. **ペアに聞く**: ペアレビュー時に相談
4. **公式ドキュメント**: Expo/Supabaseのドキュメント
5. **Perplexity**: 技術的な質問を検索
