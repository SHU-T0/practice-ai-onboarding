
| 15:00-16:00 | 夕会: フィードバック共有 | 全体 |

#### テストの観点

各アプリを30分テストして、Googleフォームで回答:

**1. 第一印象（5分）**
- [ ] アプリの目的が理解できたか
- [ ] 最初の画面はわかりやすいか
- [ ] ログイン・サインアップは簡単か

**2. 機能テスト（15分）**
- [ ] 全ての機能を試す
- [ ] データの追加・編集・削除
- [ ] エラーが発生しないか
- [ ] わかりにくい操作はないか

**3. UI/UX（5分）**
- [ ] デザインは見やすいか
- [ ] ボタンは押しやすいか
- [ ] フィードバックがあるか
- [ ] ローディングが適切か

**4. バグ・問題点（5分）**
- [ ] クラッシュしないか
- [ ] データが正しく保存されるか
- [ ] 通信エラー時の挙動

#### App Store用スクリーンショット

**必要な画像**:
- iPhone 15 Pro Max: 2796 x 1290 px（3枚以上）
- 主要機能を見せる画面

**撮影方法**:

1. **シミュレーターで撮影**
```bash
npx expo start
# iPhoneシミュレーターで起動
# Cmd + S でスクリーンショット
```

2. **実機で撮影**
   - 実機でアプリを開く
   - 音量UPボタン + サイドボタン
   - 写真アプリから取り出し

**スクリーンショット構成例**:
1. ログイン画面 or メイン画面
2. 一覧画面（データが入った状態）
3. 詳細画面 or 作成画面

#### アプリマーケティング基礎（ミニ座学）

**1. App Store最適化（ASO）の基本**

- **アプリ名**: 30文字以内、検索されやすい
- **サブタイトル**: 30文字以内、価値を伝える
- **キーワード**: 100文字以内、カンマ区切り
- **説明文**: 最初の3行が重要

**2. アプリ名の例**

| 悪い例 | 良い例 |
|--------|--------|
| メモアプリ | Tips共有 - 社内ナレッジ管理 |
| 訪問記録 | 営業ログ - 顧客訪問記録 |

**3. 説明文の構成**

```
[1行目] このアプリで何ができるか（結論）
[2-3行目] 誰のためのアプリか、どんな問題を解決するか

[主要機能]
- 機能1
- 機能2
- 機能3

[使い方]
1. ステップ1
2. ステップ2
3. ステップ3

[こんな人におすすめ]
- ペルソナ1
- ペルソナ2
```

#### Claudeで説明文を生成

```
以下のアプリのApp Store説明文を作成してください。

[要件定義書をコピペ]

【条件】

【出力形式】

- サブタイトル（30文字以内）

- 説明文（200文字）
```
