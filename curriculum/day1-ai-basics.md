### 10:00-10:15 ミニ座学: プロンプトの3原則

#### 講師向けメモ

- 次の実践セッションへの布石

#### 受講者配布用資料

**プロンプトの3原則**

1. **具体的に指示する**
   - 悪い例: 「メールを書いて」
   - 良い例: 「取引先への謝罪メールを、150文字以内で書いて」

2. **役割を与える**
   - 悪い例: 「競合分析して」
   - 良い例: 「あなたはマーケティングのプロです。競合3社の強み・弱みを表で比較してください」

3. **出力形式を指定する**
   - 悪い例: 「提案書を作って」
   - 良い例: 「提案書を以下の構成で作成してください: 1.課題 2.解決策 3.効果 4.スケジュール」

**練習プロンプト**

```
あなたは営業戦略のプロフェッショナルです。
当社の新製品「AIチャットボット」を、中小企業の経営者に提案する営業資料を作成してください。

以下の構成で、各セクション100文字程度でまとめてください:
1. 課題（中小企業が抱える顧客対応の問題）
2. 解決策（AIチャットボットでどう解決するか）
3. 導入効果（コスト削減、顧客満足度向上等）
4. 導入ステップ（3ステップで簡潔に）
```

### 13:00-13:15 ミニ座学: Cursorとは何か

#### 講師向けメモ

- VSCodeとの違いを簡潔に説明
- 実際の画面を見せながら説明
- AI設定の概要を「裏側にはこういう仕組みがある」程度に触れる
- 次のデモへの期待を高める

#### 受講者配布用資料

**Cursorとは**

Cursor = コードエディタ + AI

**できること**:

1. **Tab補完**: コードを書き始めると、AIが続きを提案
2. **Cmd+K**: 選択した部分を「日本語の指示」で書き換え
3. **Chat**: コードについて質問・相談
4. **Composer**: アプリ全体を一気に生成・編集

**従来のコーディング vs Cursor**

| 従来 | Cursor |
|------|--------|
| 構文を覚える | AIが構文を書く |
| エラーをGoogle検索 | AIに聞く |
| StackOverflowで調べる | Chatで解決 |
| 数時間〜数日 | 数分〜数時間 |

**重要**: Cursorは魔法の杖ではありません。「何を作りたいか」を考えるのは人間の仕事です。

**【参考】CursorのAI設定 — 裏側で何が起きているか**

Cursorには、AIの挙動を調整する4つの仕組みがあります。今日は概要だけ知っておけばOKです。

| 設定 | 一言で言うと | 営業に例えると |
|------|------------|--------------|
| **モデル選択** | どのAIエンジンを使うか | 案件に合わせて専門家を選ぶ |
| **Rules** | AIに「このプロジェクトのルール」を事前に伝える | 新人に引き継ぎメモを渡す |
| **Agent Mode** | AIが自律的に複数の作業を連続実行 | 「この提案書を仕上げて」と丸ごと依頼 |
| **MCP** | AIに外部サービス（DB等）へのアクセス権を与える | 営業に顧客DB検索権限を渡す |

> この研修では、Day 1の午後にCursor Settingsの「Rules」にルールを設定します。これにより、AIが常に「Expo + TypeScript + 日本語コメント」のルールに従ってコードを書いてくれます。設定は1回だけで、以降の全プロジェクトに自動適用されます。
>
> 詳細は [CursorのAI設定ガイド](../guides/cursor-ai-settings.md) を参照してください。

#### 受講者配布用手順

**ステップ0: ターミナルとは（初めて触る人向け）**

ターミナルは「文字でパソコンに指示を出す画面」です。

**営業に例えると**: マウスでの操作が「対面での会話」なら、ターミナルは「メールでの指示」です。正確に伝えれば、正確に動きます。

**ターミナルの開き方**:

1. Macの `Spotlight検索`（`Cmd + Space`）を開く
2. 「ターミナル」と入力してEnter
3. 黒い画面が開く → これがターミナルです

**基本的な使い方**:

```
コマンドを入力 → Enterキーで実行 → 結果が表示される
```

**覚えるコマンドは3つだけ**:

| コマンド | やること | 例 |
|---------|---------|-----|
| `cd フォルダ名` | フォルダに移動する | `cd ~/Documents` |
| `ls` | 今いる場所のファイル一覧を見る | `ls` |
| `npx ...` | アプリの作成や起動 | `npx expo start` |

> **重要**: ターミナルのコマンドは**半角英数字**で入力します。日本語入力がONになっていると動きません。入力前に英数キーを押してください。

**ステップ1: プロジェクト作成**

1. ターミナルを開く（上記の手順で）
2. 以下を**1行ずつ**コピー＆ペーストして実行（Enterで実行）:

```bash
cd ~/Documents
npx create-expo-app@latest hello-world-app --template blank-typescript
cd hello-world-app
```

## 補足資料

### よくある質問と回答

**Q: コードの意味が分かりません**

A: 今は分からなくて大丈夫です。AIがコードを書くので、皆さんは「何を作りたいか」を考えることに集中してください。コードの意味は、作り続けるうちに自然と分かってきます。

**Q: エラーが出たらどうすればいいですか?**

A:
1. まずエラーメッセージをコピー
2. Cursorの Chatに貼り付けて「このエラーを解決してください」
3. それでも解決しない場合は隣の人に相談
4. 最後に講師を呼ぶ

**Q: プロンプトがうまく書けません**

A: 最初は誰でもそうです。コツは「具体的に」「役割を与える」「出力形式を指定」の3つです。今日やった例を真似しながら、自分なりにカスタマイズしてみてください。

**Q: AIの回答が間違っていたらどうしますか?**

A: AIは完璧ではありません。出力を必ず自分で確認し、おかしいと思ったら「これは正しいですか?」と追加で質問してください。AIとの対話を通じて、より良い答えに近づけます。
