## 要件定義の3ステップ

### ステップ1: ワンライナー（1文で説明）

**テンプレート**:
```
[誰]が[何]するための[アプリ/サービス]
```

**例**:
- ❌ 悪い: 「便利なメモアプリ」（誰が何をするか不明）
- ✅ 良い: 「営業担当者が顧客訪問記録を写真付きで残すアプリ」

### ステップ2: ユーザーストーリー（3〜5個）

**テンプレート**:
```
[ユーザー]として、[目的]のために、[機能]がほしい
```

**例**:
```
- 営業担当者として、訪問後すぐ記録するために、写真付きメモ機能がほしい
- 営業担当者として、過去の訪問履歴を振り返るために、検索機能がほしい
- 営業マネージャーとして、チームの活動状況を把握するために、一覧表示機能がほしい
```

### ステップ3: MVP仕分け（MoSCoW法）

MoSCoW法（読み方: **モスコウ法**）は、Must / Should / Could / Won't の頭文字を取った優先順位付けのフレームワークです。

| 優先度 | 意味 | 判断基準 |
|--------|------|----------|
| **M**ust have | 必須 | これがないとアプリの価値がゼロ |
| **S**hould have | 重要 | あった方が良いが、MVP後でもOK |
| **C**ould have | あれば嬉しい | 時間があれば追加 |
| **W**on't have | 今回はやらない | 明確に除外 |

---

## MVP厳格ルール

**3週間で作るため、スコープを絞る**

### 制約条件

- **画面数**: 3つ以内
- **テーブル数**: 2つ以内
- **外部API**: 1つまで（必要な場合のみ）
- **認証**: Supabase Authのみ（SNSログインは後回し）
- **画像**: 最大1機能（カメラまたはギャラリー）

### 講師ジャッジ権限

「それはMVPから外す」と明確に判断する。理由:
- 技術的に複雑すぎる
- 3週間で完成しない
- 別機能で代替可能

## 業務自動化体験（Make.com）（15:00-16:00）

### 講師向けメモ
- Make.com Free アカウントは事前に作成済み
- デモ → 全員で同じフローを作成 → カスタマイズの流れ

### 受講者配布用手順

**Make.comとは**
Excelのマクロのように、複数のサービスを自動で連携させるツール。コード不要。

**ハンズオン: Supabaseに新データが追加されたらSlackに通知するフロー**

1. Make.com (https://www.make.com) にログイン
2. 「Create a new scenario」をクリック
3. トリガー: 「Supabase」→「Watch Rows」→ テーブルを選択
4. アクション: 「Slack」→「Send a Message」→ チャンネルとメッセージを設定
5. 「Run once」でテスト
6. 動作確認できたら「Turn on」で自動化を有効化
