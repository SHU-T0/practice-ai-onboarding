# もくもく会計画

## 概要
研修後の自走力を維持するため、毎週金曜30分のもくもく会を実施。

## スケジュール

| タイミング | 施策 | 内容 |
|-----------|------|------|
| 毎週金曜 30分 | もくもく会 | 自主制作+質問。各自の進捗共有5分+もくもく作業20分+振り返り5分 |
| 2週間後 | 業務改善アプリ企画発表 | 自分の業務で改善したいことをアプリ企画としてまとめて発表 |
| 1ヶ月後 | 個別1on1 | つまずき・不安ヒアリング。必要に応じてフォロー計画を再策定 |
| 2ヶ月後 | ミニハッカソン | 2-3人チームで半日〜1日のアプリ制作。テーマは当日発表 |
| 3ヶ月後 | 成果発表会 | 経営層向けに研修後の成果を発表 |

## もくもく会の進め方

### タイムテーブル（30分）
| 時間 | 内容 |
|------|------|
| 0:00-0:05 | 各自の進捗報告（1人1分） |
| 0:05-0:25 | もくもく作業 or 質問タイム |
| 0:25-0:30 | 振り返り + 来週の目標設定 |

### ルール
- 参加は任意だが、可能な限り出席
- カメラON推奨（集中力が上がる）
- 質問は遠慮なく（「初歩的」な質問大歓迎）
- 成果物のシェアを推奨

### もくもく会でやること（例）
- 研修で作ったアプリの改善
- 新しいアプリのアイデア出し（Claudeと壁打ち）
- 業務自動化フローの構築（Make.com）
- プロンプト集への追加
- 技術記事・チュートリアルの実践

## ミニハッカソン（2ヶ月後）企画案

### 形式
- 2-3人チーム
- 半日〜1日
- テーマは当日朝に発表

### テーマ例
- 「社内の〇〇を便利にするアプリ」
- 「お客様向けの〇〇アプリ」
- 「業務で毎日5分短縮できるツール」

### 評価基準
- アイデアの独創性
- 実用性（実際に使いたいか）
- 完成度
- プレゼン力

## KPI（フォローアップの成功指標）
| 指標 | 目標 |
|------|------|
| もくもく会の参加率 | 80%以上 |
| 2ヶ月以内に個人アプリv2をリリース | 3人以上 |
| 業務自動化フローを1つ以上構築 | 全員 |
| 3ヶ月後に新規アプリを企画 | 3人以上 |
